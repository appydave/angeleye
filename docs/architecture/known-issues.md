# Known Data-Quality Issues

Living register of open data-quality and ingestion issues in AngelEye. Each row links to a re-runnable audit script under `scripts/audits/` so any future session can confirm whether the issue is still present.

**Surfaced in-app**: the Diagnostics view (`/diagnostics`) reads the latest snapshot from `~/.claude/angeleye/diagnostics-snapshot.json` and shows live counts plus the `Open issues` list below.

**Refresh the snapshot**: `npm run audit:registry`

## How to use this doc

1. **When investigating ingestion or enrichment behaviour**, read the open issues table first. Many surprising results come from one of the items above.
2. **Before adding a new heuristic to the classifier**, check whether it depends on a field still in flight (e.g., `session_kind`).
3. **When closing an issue**, move it to the Resolved section with a date and a one-line note on how it was confirmed fixed.
4. **When discovering a new issue**, add a row with `id` (kebab-case slug â€” used by Diagnostics view), title, observation date, status, and a verification command. Add the `id` to the `OPEN_ISSUES` array in `server/src/routes/diagnostics.ts` so it surfaces in the UI.
