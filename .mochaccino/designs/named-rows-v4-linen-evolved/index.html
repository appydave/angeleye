Why this version

  • Takes the v2-linen structure you liked — fixed columns, Active/Ended grouping, clean alignment — and evolves it with the data model as it stands today.
  • Type badge in its own column. You can scan session types vertically — all the BUILDs line up, the KNOWLEDGE stands out immediately. In v1-v3 the badge was inline with the name, making it impossible to scan by type.
  • Named rows: name replaces project, project becomes context. "bmad-relay" is the hero, "supportsignal" is the quiet context after it. No pennant, no badge, no extra line — just bolder text and the amber left border.
  • Active/Ended grouping answers "what's running right now?" at a glance. The count badges give you the number without counting rows.
  • Session ID removed from view. Available on click (copy to clipboard) but not wasting space on every row.
  • Prompt stays as the second line for everyone. It's the most useful context for "what was this session about?" — more useful than a path or ID.
  • Three-dot menu (⋯) replaces the em-dash. Conventional affordance for "more actions" without cluttering the row.
