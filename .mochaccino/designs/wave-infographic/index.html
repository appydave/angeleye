WAVE STATES LEGEND

Each step in the wave can be in one of six visual states, encoded through border, opacity, and glow.

B

DONE

Green border ring. Session completed successfully with a PASS verdict.

T

ACTIVE

Amber glow with pulse animation. The currently executing session.

L

PENDING

Ghosted at 35% opacity. Not yet started; waiting for upstream steps.

N

CONDITIONAL PASS

Amber dashed border. Passed with patches — triggers a backtrack loop.

A

FAILED

Red border on light red background. Session ended with REJECT or FAIL.

S

SKIPPED

Dotted border, half opacity, strikethrough. Step was bypassed (lighter ceremony).

PROGRESSIVE DISCLOSURE STACK

Three levels of detail, each reachable by clicking into the previous. The wave view sits at Level 2.

1

SPRINT BOARD

Kanban overview of all stories in a sprint. Columns: Planned, In Progress, Review, Done. Each card shows story ID, agent chain progress, and health status.

2

WAVE VIEW

This design. The horizontal agent pipeline for a single story. Shows every session as a step with state, backtrack arcs, and the three-panel detail layout.

3

SESSION DETAIL

Full event feed for a single session. Every tool call, file edit, bash command, and LLM exchange rendered as a scrollable timeline with enrichment badges.
