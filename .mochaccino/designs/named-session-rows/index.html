Design Rationale

  • Same row height everywhere. Named and unnamed rows are identical in structure — two lines (label + prompt). Consistent rhythm means the list is scannable at any length.
  • 3px amber left border is the only structural difference. Subtle enough not to compete with focus state, visible enough to catch the eye when scrolling.
  • No "named" badge. The presence of a human-chosen name is self-evident. Labelling it is redundant.
  • Single ⚑ pennant glyph in amber before the name — a quiet signal, not a badge. Takes zero extra vertical space.
  • Project name as muted context (e.g. "supportsignal", "brains") appears inline after the name, same line, tiny and dim. No full paths — just the project slug, enough to orient without noise.
  • Name replaces project as hero text. Named rows show the name where unnamed rows show the project. The name IS the elevation — it doesn't need extra visual weight.
  • No background wash. The amber border does the work. Background changes are reserved for hover and focus states, not categories.
  • Prompt text stays on line two for both named and unnamed. This is the most useful secondary info. Never push it to line three.
